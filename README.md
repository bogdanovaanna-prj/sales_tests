Запуск тестов
1) Запустить сервис командой "go run ../sales_service/main.go"
2) Запустить все тесты "go test -v ./..."

Найденные проблемы:
1) Некорректно обрабатывается правая граница расписания
ПО ТЗ:
   Время задаётся строго в формате HH:mm.
   from включительно.
   to не включительно.
По факту: граница to включается в акцию
Тесты, которые указывают на баг:
Test_SalesService_CreateSale_HappyPath/Два_дня,_зона:_Берлин,_акция:_неактивна,_время:_точная_правая_граница_дня_1/
Test_SalesService_CreateSale_HappyPath/Два_дня,_зона:_Берлин,_акция:_неактивна,_время:_точная_правая_граница_дня_2/
2) Некорректно обрабатывается переход акции за полночь
ПО ТЗ:
   Интервал может переходить через полночь. Например, FRIDAY 22:00–02:00 означает:
   в пятницу с 22:00 до 23:59 продажа активна;
   в субботу с 00:00 до 01:59 продажа активна;
   в субботу в 02:00 продажа уже не активна.
   Несколько интервалов в один день разрешены.
По факту:
    Программа возвращает статус active для полуночи(00:00), предшествующей началу акции, хотя акция ещё не действует
    Программа возвращает статус inactive для полуночи, которая относится к периоду акции и статус должен быть active
    Программа возвращает статус inactive для времени после полуночи, в период которого статус акции должен быть active
Тесты, которые указывают на баг:
Test_SalesService_CreateSale_HappyPath/Один_день,_интервал_через_полночь,_зона:_Токио,_акция:_НЕактивна,_время:_полночь_до_начала_акции
Test_SalesService_CreateSale_HappyPath/Один_день,_интервал_через_полночь,_зона:_Токио,_акция:_активна,_время:_полночь
Test_SalesService_CreateSale_HappyPath/Один_день,_интервал_через_полночь,_зона:_Токио,_акция:_активна,_время:_за_полночь
3) Некорректно обрабатывается акция, для которой не указано расписание
ПО ТЗ:
   Пустой schedule означает, что продажа никогда не активна.
По факту: сервис возвращает ошибку 400 Bad Request
